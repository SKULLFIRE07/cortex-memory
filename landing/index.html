How It Works

Three-layer memory architecture

1

Working Memory hot

Always injected into every session. Active project state, current goals, recent decisions.

2

Episodic Memory warm

Injected when relevant. Past session summaries, resolved bugs, tried approaches.

3

Semantic Memory cold

Queryable on demand. Architecture docs, team conventions, cross-project patterns.

Features

Everything your AI needs to remember

3-Layer Memory Architecture

Hot, warm, and cold memory tiers ensure the right context is available at the right time.

Auto-Generated Decision Logs

Architecture Decision Records (ADRs) created automatically from your sessions.

Team Sync via Git

Memory is stored as files in your repo. Share context across the team with git push.

Cross-Project Intelligence

Patterns learned in one project carry over. Your AI gets smarter across your entire codebase.

Visual Memory Browser

VSCode sidebar panel to inspect, edit, and manage all memory layers in one place.
